WWPass basic authentication

The example below implements minimalistic WWPass authentication. It is based on WWPass PUID service.

PUID (Provider's User ID) is a random number which is initially generated by WWPass core network for the particular SP-User pair.

For each Service Provider (SP) PUIDs are guaranteed to be unique and different for different users. Every SP gets its own PUID set. User will rather have another PUID in another SP namespaces. However, the uniqueness is checked only inside particular SP domain.

Obviously, the PUID may serve as a pointer to the user record in a Service Provider database. Particularly, we can add a new "PUID" field into a user record. Now, instead of identifying users by

SELECT * FROM users WHERE username='$myusername' and passcode='$mypassword'

we will write

SELECT * FROM users WHERE wwpass_puid='$puid'

Preconditions:

  • You have a web server with Java support. We have tested the application on Apache Tomcat 7.0.42 - Java SE Development Kit 7 - Windows 7
  • You have registered your site and got WWPass Service Provider credentials (certificate and private key). If your site has the URL of "mysite.com" and you follow the recommended file naming when obtaining SP credentials, the files will be named as mysite.com.crt (for certificate) and mysite.com.key (for private key).

Next, we assume the following directory structure for the application

./src
    /com
       /project_name
          wwpass.java
          WWPassConnection.java
./WebContent
    wwpass.jsp
    /META-INF
    /WEB-INF
       web.xml
       /lib
          commons-codec-1.19.0.jar

 

./src/com/project_name/wwpass.java is a servlet containing application logic

./src/com/project_name/WWPassConnection.java is a library providing Java methods for authentication
./WebContent/wwpass.jsp is a JSP displaying the WWPass login button. It provides JavaScript functions for button click handling

./WebContent/WEB-ING/web.xml is a generic servlet configuration file
./WebContent/WEB-ING/lib/commons-codec-1.19.0.jar is a library providing base64 encoder.

And the following folders on the C:// drive:

./images
    loginwWWP-257x56_mouseover.png
    loginwWWP-257x56.png
./sdk-test
    Certificate.crt
    Key.key
    /_MACOSX
       _Certificate.crt
       _Key.key

./images directory holds "Login with WWPass" button images
./sdk-test directory holds your SSL certificate and a key

NOTE:The code intentionally lacks error checking for the sake of simplicity and clarity.

You will need to configure:

  • wwpass.java with your Service Provider name and certificate file locations
  • wwpass.jsp if you changed the location of /images folder
